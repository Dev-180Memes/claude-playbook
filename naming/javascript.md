# JavaScript Naming Conventions

- `camelCase` for variables, functions, methods, and parameters
- `PascalCase` for classes and constructor functions
- `SCREAMING_SNAKE_CASE` for module-level constants
- Boolean variables and functions should read as yes/no questions: `isReady`, `hasPermission`, `canSubmit`
- Event handler functions should be prefixed with `on` or `handle`: `onSubmit`, `handleClick`
- Avoid generic function names like `callback`, `fn`, `func` — name them for what they do: `onSuccess`, `parseRow`
